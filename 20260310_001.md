## 보드 구조

### 기본 정보
- 전체 판: 2차 배열 `Cell[][]`
- 열(col): 7 고정
- 행(row): 골드로 묶음 추가
- 각 셀: 정사각형

### 셀 블록 타입

| 약어 | 이름 | 설명 |
|------|------|------|
| RS | Rail Start | 레일 시작점 |
| RE | Rail End | 레일 종료점 — 아이템 판매 |
| RL | Rail Left | 레일 좌 |
| RR | Rail Right | 레일 우 |
| RU | Rail Up | 레일 위 |
| RD | Rail Down | 레일 아래 |
| PR | Production | 생산 — 자동 생산 후 레일로 배출 |
| FA | Factory | 공장 슬롯 — 배치 후 타입 선택 (WA/PA/PK) |
| CR | Crane | 크레인 — 아이템 재투입 (양방향) |
| EM | Empty | 빈블록 |

### Factory 속성
```
Factory (FA)
├── type: WA | PA | PK   ← 배치 후 선택 (세척/가공/포장)
├── level: number        ← 골드로 업그레이드
└── animal: Animal       ← 배치된 동물 (능력 강화)
```

## 묶음 확장 (골드로 추가)

묶음 추가 시 이전 묶음 B의 `RE` → `RD` 로 변경 후 새 묶음 2행 추가

```typescript
// 묶음 2 추가 후
['EM', 'PR', 'EM', 'PR', 'EM', 'PR', 'EM'],  // 생산행 0
['RS', 'RR', 'RR', 'RR', 'RR', 'RR', 'RD'],  // 생산행 1
['EM', 'FA', 'EM', 'FA', 'EM', 'FA', 'RD'],  // 묶음 1 - A
['RD', 'RL', 'RL', 'RL', 'RL', 'RL', 'RL'],  // 묶음 1 - B (RE→RD)
['RD', 'FA', 'EM', 'FA', 'EM', 'FA', 'EM'],  // 묶음 2 - A
['RR', 'RR', 'RR', 'RR', 'RR', 'RR', 'RE'],  // 묶음 2 - B
```

### 아이템 흐름
```
PR(생산) → RS → RR... → RD
                         ↓
              FA FA FA ← RD
                         ↓
         RE(판매) ← RL...
```

## UI 컴포넌트 (TDS 가이드 적용)

> 참고: DESIGN_GUIDE.md

### Navigation (최상단 필수)
- 위치: 화면 최상단 고정
- 내용: 서비스명 "동물노동조합" 표시
- TDS Navigation 컴포넌트 스타일 적용

### 묶음 추가 버튼 (BottomCTA 스타일)
- 위치: 보드 하단
- 가로: 전체 너비 (`cellSize * 7`)
- 세로: 2칸 (`cellSize * 2`)
- 텍스트: "묶음을 추가해요" (해요체)
- 동작: 마지막 행 `RE` → `RD` 변경 후 새 묶음 2행 추가

### 셀 크기
- 모바일 대응: `cellSize = Math.floor(window.innerWidth / 7)`
- 화면 리사이즈 시 동적 재계산
- 각 셀은 정사각형 (`cellSize * cellSize`)

### 전체 레이아웃
- 기준 너비: 375px
- 배경색: 라이트 모드 기준
- 텍스트: 해요체 사용
