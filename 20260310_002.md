# 동물노동조합 - 실행계획 20260310_002

## 목표
아이템 흐름 구현 (PR 생산 → 레일 이동 → RE 도달 시 제거)

> FA 수집은 003에서 구현

---

## 공통 설정 (config.ts)

| 설정 | 값 | 설명 |
|------|-----|------|
| PRODUCE_INTERVAL | 250ms | PR 아이템 생산 주기 (테스트: 4배 생산) |
| MOVE_SPEED | 1200ms | 레일 한 칸 이동 시간 (느리게) |
| PICK_TIME | 800ms | FA가 아이템을 집는 시간 |
| ITEM_SIZE_RATIO | 0.7 | 아이템 크기 (cellSize * 0.7) |

---

## 아이템 데이터 구조

```typescript
interface Item {
  id: string
  x: number       // 현재 위치 (보드 기준 픽셀)
  y: number
  dx: number      // 이동 방향
  dy: number
  targetX: number // 다음 셀 중앙 (waypoint)
  targetY: number
}
```

---

## 아이템 흐름 규칙

- 아이템은 **Waypoint 기반** 이동 — 셀 중앙을 찍고 방향 전환 (대각선 이동 없음)
- 아이템 크기: `cellSize * 0.7` (원형)
- 충돌 감지: 현재→타겟 벡터 기준으로 앞에 있는 아이템과 거리 < `itemSize` 이면 대기
- PR 3개가 각각 250ms마다 RS에 스폰 시도
- RS가 1개라 실질 생산 속도 = `MOVE_SPEED * ITEM_SIZE_RATIO` ≈ 840ms에 1개

## 흐름 순서

```
PR (PRODUCE_INTERVAL마다 스폰 시도)
  → RS (스폰 위치)
  → RR/RL/RU/RD (레일 따라 Waypoint 이동)
  → RE (도달 시 제거)

※ FA 수집: 003에서 구현
```

---

## 주요 결정 사항

- RS가 막혀 있으면 스폰 안 됨 (중복 불가)
- 뒤에서 오는 아이템이 앞 아이템을 막는 데드락 방지: 충돌 방향을 `현재→타겟` 벡터 기준으로 판단
- RE 제거: 아이템이 RE 셀 중앙에서 `cellSize * 0.3` 이내일 때 제거
- 현재 생산 속도 구조 유지 (RS 1개 기준 자연스러운 병목)
