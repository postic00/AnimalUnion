## Producer 설계

```typescript
interface Producer {
  row: number
  col: number
  level: number  // 0 = 비활성, 1+ = 생산
}
```

### config 전역 설정
| 키 | 값 | 설명 |
|---|---|---|
| PRODUCER_VALUE_MULTIPLIER | 2 | 레벨당 골드 배수 |
| PRODUCE_INTERVAL | 10000 | 생산 주기 기본값 (ms) |
| PRODUCE_INTERVAL_MULTIPLIER | 1 | 생산 주기 감소 배수 |
| PRODUCER_UPGRADE_BASE | 1 | 업그레이드 비용 기본값 |
| PRODUCER_UPGRADE_EXPONENT | 2 | 업그레이드 비용 지수 |

### 수식
- 아이템 가치: `PRODUCER_VALUE_MULTIPLIER^(level-1)`
- 생산 주기: `PRODUCE_INTERVAL / (level * PRODUCE_INTERVAL_MULTIPLIER)`
- 업그레이드 비용: `PRODUCER_UPGRADE_BASE * (level+1)^PRODUCER_UPGRADE_EXPONENT`

### 레벨별 수치
| level | interval |
|-------|----------|
| 0 | 비활성 |
| 1 | 10초 |
| 2 | 5초 |
| 3 | 3.33초 |
| 5 | 2초 |
| 10 | 1초 |

## Clicker 설계

```typescript
interface Clicker {
  clickCount: number  // 누적 클릭 수
  threshold: number   // 아이템 1개 생산에 필요한 클릭 수 (기본 2)
}
```

- 위치: 탭 우측상단 플로팅 버튼
- 클릭 2회 누적 → value=1 아이템 1개 RS에 스폰
- 수치(threshold, value)는 추후 config로 관리
- 업그레이드(level 등)는 나중에 추가
- 링 진행 표시: SVG 원형 프로그레스 (애니메이션 없음, 즉시 반영)

- 핸들바 제거

- 초기 gold: 100 (테스트용)

## balance.ts 수식

### 라인 확장 비용
```
getBundleCost(count) = floor(BASE * (count+1)^EXPONENT)
BASE = 1, EXPONENT = 2**2 = 4

count 0 → 1
count 1 → 16
count 2 → 81
```

### PR 업그레이드 비용
```
getProducerUpgradeCost(level) = floor(BASE * (level+1)^EXPONENT)
BASE = 1, EXPONENT = 2**2 = 4

level 0 → 1
level 1 → 16
level 2 → 81
```

### PR 생산 주기
```
getProducerInterval(level) = PRODUCE_INTERVAL / (level * MULTIPLIER)
PRODUCE_INTERVAL = 10000ms, MULTIPLIER = 1

level 1 → 10초
level 2 → 5초
level 5 → 2초
level 10 → 1초
```

### PR 아이템 가치
```
getProducerValue() = ITEM_BASE_VALUE = 1 (고정, FA 처리로 증가 예정)
```

### 클릭커
```
getClickerThreshold() = CLICKER_THRESHOLD = 2 (클릭 2회 → 아이템 1개)
```
