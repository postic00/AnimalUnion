## 재료 탭 (아이템 탭 → 재료 탭)

### 개요
- 레일 위를 흐르는 아이템 = 마라탕후루 재료
- 재료 탭: 골드로 수량 레벨업 (1→2→4→8... 2배씩)
- 환생 탭: 포인트로 가치 레벨업 (기존 유지)
- 아이템에 quantity 필드 추가, 레일 위에 수량 표시 (x2, x1.23k 등)

### 재료 등급 (Product 리스트)

### 수량 포맷
- 1000 미만: 그대로 표시 (x1, x8...)
- 1000 이상: 3자리 포맷 (x1.23k, x12.3k, x123k)
- `formatQuantity` 함수 추가 (`formatGold.ts`에 같이)

### 수량 레벨업

## 동물 배치 UI 개선

### 동물 행 레이아웃 (환생탭)
```
[이름] [+N%] [Lv.N] [배치/회수] [업그레이드]
```
- 미배치: **배치** 버튼 → 클릭 시 배치 모드 진입
- 배치중: **회수** 버튼 → 클릭 시 해당 공장에서 동물 제거

### 배치 흐름
1. 동물 행의 **배치 버튼** 클릭
2. 바텀시트 닫히고 보드 표시
3. 보드의 FA 셀 하이라이트 + 클릭 가능 상태
4. FA 셀 클릭 → 해당 공장에 동물 배치
5. 취소 버튼으로 모드 종료

### 회수 흐름
1. 동물 행의 **회수 버튼** 클릭 → 즉시 해당 공장에서 동물 제거

### 상태 관리
- `placingAnimalId: AnimalId | null` — 현재 배치 중인 동물 ID (전역 상태)
- FA 셀: placingAnimalId가 있을 때 클릭 가능 + 하이라이트

### 구현 항목
- PrestigeTab 동물 목록에 배치/회수 버튼 추가
- App.tsx에 placingAnimalId 상태 추가
- Board/Cell에 FA 셀 클릭 핸들러 추가
- FA 셀 하이라이트 스타일 추가

## 수량 메커니즘 확정 (구현 완료)

### 핵심 규칙
- **수량 = 박스 1개를 만드는 데 필요한 원자재 생산 횟수**
- quantity=1: 생산 1번 → 박스 1개 출력
- quantity=2: 생산 2번 → 박스 1개 출력 (2배 가치)
- quantity=4: 생산 4번 → 박스 1개 출력 (4배 가치)

### 구현 방식
- `useGameLoop.ts`에 `produceCountersRef: Record<string, number>` 추가
- PR 타이머 발동 시 카운터 증가, 카운터 >= quantity 되면 박스 스폰 + 리셋
- 박스의 `item.quantity` = 수량값 (getFinalGold에서 곱산)
- 클릭커 스폰은 별도 처리 (즉시 스폰, quantity 값 그대로 가치 반영)

### 재료 등급별 독립 관리
- `materialQuantityLevels: number[]` (20개 배열, 초기값 전부 1)
- 환생 시 리셋됨
- 재료 탭에서 등급별 개별 업그레이드
