## 1. FA 처리 시간 개선

### 현재 문제
- `getFactoryPickTime(level)` — 수량 무관, 레벨만 반영
- quantity=1 박스와 quantity=100 박스가 동일한 처리 시간

### 변경 후 공식
```
총 처리 시간 = 2 × pickTime + processTime

pickTime    = FA_PICK_TIME (고정, 잡기 = 놓기 동일)
processTime = FA_PROCESS_TIME_BASE × quantity / (level × FA_LEVEL_EFFICIENCY)
```

### config 상수 분리
```typescript
FA_PICK_TIME: 200            // 잡기/놓기 시간 (고정, ms)
FA_PROCESS_TIME_BASE: 800    // 처리 기본 시간 (ms)
FA_LEVEL_EFFICIENCY: 1       // 레벨당 처리속도 증가 배수
```

### 원칙
- 처리 시간 수식과 레벨업 비용 수식은 별도 관리 (연동 금지)
- 밸런스 수치 변경 시 독립적으로 조정 가능

## 2. 공장 타입별 역할

### WA (세척)
- 등급 변경 없음
- 아이템 **가치 증가** (가격 배수)
- 선택적 배치 — 없어도 PA 진행 가능
- 배치 전략: 고가치 아이템 경로에 배치할수록 유리

### PA (가공)
- **레시피 기반 조합** → 등급 상승
- 내부 버퍼에 재료 수집 → 레시피 충족 시 출력
- 출력 quantity = `getMaterialQuantity(materialQuantityLevels[outputGrade - 1])`
- 입력 필요량 = `recipe_count × outputQuantity` (출력 수량 비례)

### PK (포장)
- 완성품 포장 → 최종 등급 출력 (마라탕, 탕후루, 마라탕후루)

---

## 3. 생산기 (PR) 타입 분리

- PR은 건설 시 생산할 재료 타입 선택
- **PR1** → 고추 (grade 1) 생산
- **PR2** → 양파 (grade 2) 생산
- **PR3** → 마늘 (grade 3) 생산

---

## 4. 아이템 등급 상승

### 현재 문제
- FA 통과 시 waBonus/paBonus/pkBonus만 누적
- grade는 항상 1 고정 → itemValueLevels[0]만 사용됨

### 변경 후
- PA/PK 통과 시 grade 실제로 변경
- 출력 아이템 가치 = `GRADE_BASE_VALUES[outputGrade-1] × getItemValue(itemValueLevels[outputGrade-1])`
- WA는 grade 변경 없이 가치 배수만 적용

### 등급별 기본 가격 (balance.ts)
```typescript
GRADE_BASE_VALUES: [1, 3, 6, 10, 20, 35, 55, 80, 120, 180, 250, 350, 500, 700, 1000, 1400, 2000, 3000, 5000, 10000]
```

## 6. 버퍼 시스템

### RS 버퍼
- PR 생산 → RS 버퍼에 적재 → 컨베이어 투입
- 버퍼가 크면 생산 효율 향상

### FA 버퍼
- 모든 공장 공통 입력/출력 버퍼 보유
- 실제 버퍼 용량 = `FA_BUFFER_BASE + bufferLevel × FA_BUFFER_PER_LEVEL`

### config 상수
```typescript
RS_BUFFER_BASE: 3
RS_BUFFER_PER_LEVEL: 2
FA_BUFFER_BASE: 5
FA_BUFFER_PER_LEVEL: 5
BUFFER_UPGRADE_COST_BASE: 1
BUFFER_UPGRADE_COST_EXPONENT: 2
```

### 환생탭 UI
```
환생탭
├── 아이템 가치 ▼
├── 동물 ▼
└── 공장/레일 버퍼 ▼
    ├── RS 버퍼 Lv.N  ⭐[비용]
    └── FA 버퍼 Lv.N  ⭐[비용]
```
