## 1. BottomSheet 구조 개선

### 문제
탭 헤더가 콘텐츠와 함께 스크롤되어 고정되지 않음

### 해결
BottomSheet를 `.sheetHeader`(고정) + `.sheetBody`(스크롤) 구조로 분리

```tsx
// BottomSheet.tsx
export default function BottomSheet({ open, header, children }: Props) {
  if (!open) return null
  return (
    <div className={styles.sheet}>
      {header && <div className={styles.sheetHeader}>{header}</div>}
      <div className={styles.sheetBody}>{children}</div>
    </div>
  )
}
```

```css
.sheet     { display: flex; flex-direction: column; height: 40vh; overflow: hidden; }
.sheetHeader { flex-shrink: 0; padding: 14px 16px 10px; border-bottom: 1px solid #f3f4f6; }
.sheetBody   { flex: 1; overflow-y: auto; padding: 10px 16px 24px; }
```

### 탭별 헤더
- 생산/공장/재료/설정: `<h2>` 제목만
- 환생 탭: 제목 + 현재 포인트 (`⭐ {formatGold(prestigePoints)}`)
- 탭 열릴 때 `window.scrollTo({ top: document.body.scrollHeight, behavior: 'smooth' })` 로 자동 스크롤

## 4. 클리커 시스템

### 아키텍처
```
클릭 → grade 결정(첫 클릭) → threshold 계산 → 링 채우기 → 100% 도달
                                                          → 250ms 잠금
                                                          → 다음 클릭 → 스폰
```

### threshold 공식
```ts
getClickerThreshold(quantity, clickerLevel) = max(1, ceil(quantity / 2^(level-1)))
```
- 클리커 레벨 1: threshold = quantity (예: quantity=8 → 8번 클릭)
- 클리커 레벨 4: threshold = ceil(quantity/8) (예: quantity=8 → 1번 클릭)

### 자동 스폰 버그 수정 (`spawnUnlockTimeRef`)
빠른 연속 클릭 시 링이 100%가 되는 클릭과 스폰 클릭이 같은 배치로 처리되어 "자동 스폰"처럼 보이는 현상 수정

```ts
// 링이 막 채워질 때
if (next >= threshold) spawnUnlockTimeRef.current = now + 250

// 스폰 시도 시
if (clickCount >= threshold) {
  if (now < spawnUnlockTimeRef.current) return prev  // 잠금 중 무시
  // 스폰 처리
}
```

### 링 100% 시 시각 피드백 (pulse 애니메이션)
```css
.clickerButtonFull {
  animation: clickerPulse 0.7s ease-in-out infinite;
}
@keyframes clickerPulse {
  0%, 100% { box-shadow: 0 4px 16px rgba(49, 130, 246, 0.45); }
  50%       { box-shadow: 0 4px 28px rgba(49, 130, 246, 0.85), 0 0 0 6px rgba(49, 130, 246, 0.25); }
}
```

### 기타 클리커 사양
- 등급: 건설된 PR 중 랜덤 선택, 없으면 1등급
- 탭바 아이콘: 클릭 시 선택 등급 이모지로 변경, 0%가 되면 👆 복귀
- 비용: `floor(PRODUCER_UPGRADE_BASE × (level+1)^PRODUCER_UPGRADE_EXPONENT)` (생산기와 동일)
- 클릭당 기여: `2^(level-1)` (1, 2, 4, 8...)
- 클리커 업그레이드 카드: 생산 탭 최상단에 추가

const [resetKey, setResetKey] = useState(0)
// 하드리셋, 환생(포인트리셋), 환생(포인트유지) 모두:

// Board
