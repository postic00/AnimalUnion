- `versionCode`: 2 → 3
- `versionName`: 1.1 → 1.2

### 아이템 이동 간격
- `ITEM_GAP_RATIO: 0.2` 추가 (기존 ITEM_SIZE_RATIO는 시각 크기 전용 유지)
- `useGameLoop.ts`에서 이동 블로킹 거리에 `ITEM_GAP_RATIO` 사용

### config.ts 추가 값
- `CURRENT_WEEK: 1`
- `NEXT_WEEK: 1`
- `NEXT_WEEK_RATE: 0.7`
- `WEEK_START_DATE: '2026-03-10'`
- `WEEK_END_DATE: '2026-03-20'`
- `as const` 제거 → 서버 config로 런타임 오버라이드 가능
- `applyWeekConfig()` 함수 추가

### 신규 파일
- `src/lib/userProfile.ts`: 클라우드 저장/불러오기, 날짜 기반 주차 config 조회
- `saveLoad.ts`: `getDeviceId()`, `saveWeekConfig()`, `loadWeekConfig()` 추가

### 주차 config 흐름
1. 앱 시작 → localStorage week config 즉시 `applyWeekConfig()`
2. Supabase `game_config`에서 오늘 날짜 기준 조회 (`start_date <= today <= end_date`)
3. 서버 값 localStorage 저장 → `applyWeekConfig()` 재적용

### 설정 화면 (SettingsTab)
- 클라우드 저장 버튼 추가 (`onCloudSave`)
- 클라우드 불러오기 버튼 추가 (`onCloudLoad`)
- 기본 저장은 항상 localStorage, 버튼 클릭 시에만 Supabase 저장/로드

### MAU
- `device_id` (UUID, localStorage 영구 저장) 기준으로 고유 유저 식별
- `last_online_at` 기준 MAU 쿼리:
```sql
SELECT COUNT(DISTINCT device_id) FROM user_profiles
WHERE last_online_at >= NOW() - INTERVAL '30 days';
```
