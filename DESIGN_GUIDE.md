# 디자인 가이드 - Apps in Toss (TDS)

참고: https://developers-apps-in-toss.toss.im/design/overview.html

---

## 핵심 원칙

- **TDS(Toss Design System) 컴포넌트** 사용 권장
- **캔버스 기준 너비**: 375px
- **다크모드**: 현재 미지원 (라이트 모드 기준)

---

## 필수 규칙

- 모든 화면 최상단에 **Navigation 컴포넌트** 반드시 사용
- 컴포넌트 속성은 우측 패널에서만 수정
- 대부분의 TDS 컴포넌트는 **내부 패딩 포함** → 컴포넌트 간 별도 간격 불필요

---

## 핵심 컴포넌트 11개

| 컴포넌트 | 설명 |
|----------|------|
| Navigation | 모든 화면 최상단 필수 — 서비스명 표시 |
| Button | 액션 트리거 — 폼 제출, 다이얼로그, 삭제 등 |
| BottomCTA | 하단 고정 버튼 영역 |
| Badge | 상태/수량 표시 |
| Border | 구분선 |
| Asset | 이미지/아이콘 |
| ListRow | 리스트 항목 |
| ListHeader | 리스트 헤더 |
| Paragraph | 본문 텍스트 |
| Tab | 탭 네비게이션 |
| Top | 상단 영역 |

---

## UX 라이팅 원칙

- **해요체** 사용 ("했어요" O, "됐어요" X)
- **능동형** 표현 우선
- **긍정형** 표현 우선 ("있어요" O, "없어요" X)
- 과도한 경어 지양 — 친근한 말투

---

## 동물노동조합 적용 방향

### 변경 필요 사항
1. **Navigation** 상단 추가 — 서비스명 "동물노동조합" 표시
2. **배경색** — 다크 → 라이트 (TDS 라이트 모드 기준)
3. **묶음 추가 버튼** — BottomCTA 스타일 적용
4. **텍스트** — 해요체로 변경 ("+ 묶음 추가" → "묶음을 추가해요")
5. **기준 너비** — 375px 고정 기준으로 레이아웃 설계
